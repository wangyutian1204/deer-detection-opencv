# 基于OpenCV的鹿群动态识别计数系统

## 项目简介
这是我在《机器学习》课程大作业中独立完成的项目，针对野生动物监测中鹿群人工计数效率低的痛点，基于Python+OpenCV实现了一个鹿群动态识别计数系统，支持视频/摄像头实时检测、框选计数和结果导出。

## 技术栈
- 编程语言：Python 3.9
- 核心库：OpenCV、NumPy、Pillow、Tkinter
- 核心算法：帧差法、高斯模糊、轮廓检测

## 项目功能
1.  视频检测：支持MP4/AVI格式视频上传，单次播放，实时框选移动鹿群并计数；
2.  摄像头检测：调用本地摄像头，实时检测移动目标；
3.  GUI界面：基于Tkinter开发可视化界面，支持管理员/普通用户双权限；
4.  结果导出：支持检测报告TXT格式导出。

## 项目效果
- 系统匹配25FPS原始帧率，单帧处理耗时＜30ms；
- 移动目标检测响应延迟≤1帧；
- 经20+项测试用例验证，连续运行1小时无崩溃。

## 遇到的问题与解决方法
1.  **视频卡顿问题**：初期用了复杂的背景差分法，导致每帧处理时间过长，后来改用轻量化的帧差法，解决了卡顿问题；
2.  **小目标漏检问题**：初始轮廓面积阈值设置过大，小目标被过滤，后来调整了阈值，解决了漏检问题；
3.  **界面组件冲突问题**：检测过程中界面更新与视频读取冲突，后来用了`after()`方法定时更新界面，解决了画面闪烁问题。

## 后续优化方向
目前系统受帧差法算法限制，只能检测移动鹿群，无法识别静止鹿群，也无法精准区分鹿与其他移动目标。后续计划：
1.  把算法替换为YOLOv8深度学习模型，提升识别精度；
2.  引入DeepSORT多目标跟踪算法，解决重叠鹿群重复计数的问题；
3.  扩充鹿群数据集，覆盖不同光照、背景的样本。

## 运行说明

2.  运行程序：`python wildlife_monitor.py`
